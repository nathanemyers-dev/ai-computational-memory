# Version 1.4 Release Notes  

Version 1.4 strengthens the publication layer, introduces the glossary and references sections, and finalizes the structural alignment required for external consumption. This release completes the documentation spine, clarifies routing, and prepares the corpus for industry and regulatory review.

## New Artifacts

**Glossary**  
Defines the governed terminology used across the category. Establishes consistent language for continuity, supervised persistence, retention boundaries, and advisory‑risk controls.

**References**    
Introduces the formal reference structure for regulatory sources, technical sources, industry case studies, and internal versioned artifacts. Aligns the white paper with external standards and supervisory expectations.

**Structural Changes**    
The publication layer is now complete. The glossary and references sections have been added to the conceptual spine and routed in index.md. The documentation structure now mirrors regulated technical frameworks and industry standards.

**Routing Alignment**    
Index.md has been updated to include glossary.md and references.md in the correct conceptual location. Version headers have been normalized. Legacy routing inconsistencies have been corrected.

**Continuity Clarification**    
Adds the formal distinction between continuity, continuity layer, continuity failure modes, and continuity loss. Clarifies how supervisory gates, retention boundaries, and interpretation heuristics interact to prevent advisory state formation.

**Regulatory Alignment**    
Strengthens the mapping between computational memory and external regulatory frameworks. Aligns terminology with NIST AI RMF, ISO/IEC 42001, EU AI Act, OCC supervisory expectations, and UK AI Safety Institute publications.

**No Breaking Changes**    
All v1.3, v1.2, and v1.1 artifacts remain intact. No API surfaces were modified. No backward compatibility issues were introduced.
